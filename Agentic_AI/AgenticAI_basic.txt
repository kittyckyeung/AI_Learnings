AI researchers aspire to provide agents with such capabilities using data collection tools.

In a broader sense, we refer to sensors as the means by which agents receive input from their environment.

Sensors allow them to gather information by scraping the web, accessing APIs, interpreting images from cameras, capturing audio through microphones, or analyzing data from IoT devices.

Essentially, sensors enable AI agents to perceive their surroundings in much the same way human senses do.

This gathered information is then processed by the agent, allowing it to make sense of its environment and take appropriate actions.

In the case of self-driving cars, the input sensors typically include cameras, lidar sensors, GPS signals, and a reliable internet connection to the manufacturer's cloud infrastructure.

For an online shopping AI agent, however, the sensors involve web scrapers, collecting product information, APIs, accessing user browsing history modules, performing price comparisons and integrations, fetching real time weather data.

Therefore, AI agents collect input from the environment using various types of sensors.

The model layer helps agents interpret information and is responsible for knowing, but knowing and understanding information isn't enough.

When we use an LLM as an AI agent's model, the distinction between model and decision logic can blur.

Typically, the LLM can understand and reason internally.

It can also generate and select appropriate actions.

But is such a separate logic layer needed? It depends.

In many practical implementations, a thin external logic layer is useful for extracting structured actions and ensuring the llms actions are safe and legal.

This gives AI agent Deployers a bit more safety about the agent's agents actions, they'll always need to conform to the external logic layer.

In other instances, instead of a separate decision logic layer, you can use prompt engineering and instruct the LLM how to perform.

The AI agent utilizes actuators, which are mechanisms that allow the agent to interact with the environment.

In some instances, there can be physical actuators like a robot's arm or the steering system of a self-driving car, and other times the agent relies on virtual actuators, such as an API used to execute stock market orders.

The first type of agent structure will consider is the simple reflex agent.

As one can infer from its name, this is a fundamental tool that reacts directly based on immediate perception.

It doesn't have any internal memory or sophisticated models.

A good example of a simple reflex agent is a thermostat.

The goal of this device is to maintain a fixed temperature, but it isn't capable of learning and adapting.

It can't learn to use electricity more efficiently at certain times of day.

In this case, the simple reflex agent is considered an agent because it perceives its environment and then acts upon that environment. indicating a lack of internal reasoning.

Given that no internal reasoning is occurring and responses don't improve over time, it's safe to conclude that this is not an AI agent, but simply an agent.

===============================================

The next agent type we'll explore is the model based reflex agent.

Like simple reflex agents, these agents perceive their environment and act upon it.

But there's a key distinction.

Model based reflex agents use memory to maintain an internal representation of the environment.

They construct a model of the world around them.

A good example of a model based agent is a robot vacuum cleaner.

So is this an AI agent? Yes.

A model based reflex agent typically qualifies as a simple form of AI because it incorporates a model that enables the agent to understand its environment better.

But this is a simple form of AI because the robot doesn't necessarily learn or adapt based on experience,

its internal model is usually static or updated through fixed rules rather than adaptive learning algorithms.

There's no authentic learning because the robot follows a static set of rules and doesn't improve its strategy based on experience.

It simply keeps track of areas it has already cleaned and updates its internal model based on sensor input.

========
To say we have sophisticated AI capable of authentic learning, we would need a robot that can adapt through experience, discovering more efficient cleaning patterns and areas that are more polluted to begin with.

Suppose your vacuum cleaner notices, through repeated experiences, that the kitchen floor gets dirtier faster than the hallway.

A genuinely learning vacuum cleaner would start prioritizing the kitchen in its subsequent cleaning runs, because it wouldn't know whether you would provide sufficient cleaning time for both the kitchen and hallway.

This is an excellent example of how goal based agents work.
===============================================

Whenever you use Google Maps or Waze, the application compares the estimated time it will take to reach your desired destination via different routes, and then recommends the fastest one.

They search for action sequences that achieve their goal and plan these actions.

This type of search and planning didn't occur with model based and straightforward reflex agents.

For this reason, goal based agents are a step ahead in terms of AI sophistication.

At the same time, goal based agents often overlook certain subtleties, focusing exclusively on optimizing for goal completion.

At times, Google Maps may recommend routes that are too narrow, barely drivable, or otherwise difficult to navigate.

Illustrating how goal based agents can optimize for their objectives while overlooking practical nuances or user comfort.

===============================================

The next type of agent we'll consider is a utility based agent.

The reason is that utility based agents consider multiple factors to evaluate utility, rather than just achieving a predetermined goal.

It's one thing to try to find and invest in financial securities that offer an annual rate of return above 5%.

That's a goal based mechanism, but a 5% return doesn't paint the whole picture.

A competent investment advisor would also consider such factors as investment volatility, liquidity, and default risk.

These additional factors significantly affect the overall attractiveness or utility of an investment.

Utility based agents carefully evaluate these trade offs, assessing multiple criteria simultaneously, which enables them to make more nuanced and informed decisions.

===============================================
Learning agents possess the same capabilities as utility based and goal based agents, but are unique in their ability to learn.

New experiences are added to their initial knowledge base, and as the agent gains experience, this enhances its ability to operate in unfamiliar environments and improve performance.

In addition to the familiar components, environment, sensors, internal model, decision logic, and actuators, this system also includes a learning element which enhances the model over time based on feedback.

With this learning element and the mechanics built into the system, the agent is capable of self refining.

It can adapt and improve its capabilities over time.

Learn from Human? 

As previously noted, AI agents are autonomous in their decision making process.

However, they require goals and predefined rules created by humans.

One fundamental concept highlighting human influence throughout an AI agent's life cycle is known as human in the loop hittle. (HITL)

Hittle emphasizes the continuous involvement of humans in the training, monitoring, and refinement of AI systems.

The drawback is that although human feedback is highly valuable, obtaining it can be costly and time consuming.

In our customer support chatbot example, when the agent interacts with clients, it records conversations continuously, gaining new insights.

This information, along with user feedback on service quality and the user's individual preferences, helps the agent adapt and improve its performance over time.

But it takes a significant amount of time to learn from humans, and this type of learning can be rather costly.
===============================================

Learn from External System?

One of the most exciting aspects of AI agent systems is their ability to learn and continuously improve over time.

To achieve this, they rely on three key sources one.

1. Learning from other agents 
2. connecting to external sources of information such as the internet
3. human feedback.

Multi-agent feedback can be instrumental in minimizing the time that human users spend providing direction.

When one of these AI agents encounters a difficult task, such as an error experienced by a user for the first time, it identifies an effective solution through interaction with the client.

Then, this agent can share the solution with other connected agents. The next time another agent faces a similar query, it can confidently offer a proven solution without needing human intervention.

Collaborative agent to agent learning can significantly speed up response times. 

Another meaningful way in which agent systems can learn is by obtaining information from external sources to which they have access.

These external sources can include external datasets, web searches, and APIs that create a bridge with an external system.

After the missing information is retrieved from these tools, the agent can update its knowledge base and perform a genetic reasoning to help illustrate this process.

Agentic AI vs ChatGPT? 

This is one of the crucial distinctions between AI agents and AI tools like ChatGPT.

The sharing of information enables AI agents to be more general purpose than traditional static AI interfaces that have a knowledge cutoff date.

The short answer is ChatGPT is powerful but mostly reactive, while Agentic AI is proactive, autonomous, and designed to get things done on its own.

Agentic AI can:
- Break a big problem into smaller steps.
- Access tools, apps, or APIs on its own.
- Make decisions in real-time.
- Execute actions without constant human input.

For example:
- If you ask ChatGPT to plan a trip, it will give you a list of hotels and itineraries.
- An Agentic AI can not only suggest the plan but also book your flights, reserve the hotel, and send you reminders, all while staying within your budget.


- ChatGPT is perfect for brainstorming, creativity, and on-demand answers.
- Agentic AI is perfect for task execution, automation, and long-term projects.

Agent Customer Service MAS for an E-Commerce Platform
Agent 1 - Customer Support Responder (L1)

Role: Handle initial customer queries and generate helpful responses using company policies and product information.

Delivery: A first-draft response that answer the customer's question and includes relevant policy or product information.

Context: Customer message + order information + company help documentation.

Tools: 

Order Database lookup

Knowledge Base Search

Backstory/Prompt: "You are a customer support agent for an e-commerce platform. Your job is to carefully understand customer issues and provide clear, helpful responses using official documentation. If information is missing, ask clarifying question before responding."

Agent 2 - Support Quality Reviewer (L2)

Role: Review and refine response generated by L1 to ensure it is accurate, professional and complete.

Delivery: A polished final reply that improves clarity, tone and correctness before sending to the customer.

Context: L1 draft response + original customer message + company policy.

Tools: None.

Backstory/Prompt: "You ate a senior customer quality analyst. Your role is to review responses generated by the support agent and improve them. Ensure the message is polite, accurate and fully addresses the customer's concern."

Collaboration Flow

Customer submits query -> L1 agent retrieves relevant information and draft response -> L2 agent reviews and improve the response -> Final response sent to the customer.

Why only these tools

For L1 agent - to retrieve customer order information and company documentation.
For L2 agent - None needed, It focuses only on reviewing quality and completeness of the response.

Scoping
If customer queries increases, add another Agent to handle routing/prioritization or policy/billing edge-cases.
